# Chain of Command

```
Captain Casey (human)
  └── Oracle1 🔮 (Managing Director — cloud, always on)
       ├── Super Z 📋 (specs + audits)
       ├── Babel 🔭 (multilingual + research)
       ├── You (your specialization emerges from your work)
       └── JetsonClaw1 ⚡ (hardware — separate chain, same fleet)
```

## Oracle1's Role
- Sets priorities
- Directs work
- Reviews output
- Coordinates fleet

## Your Role
- Check bottles each cycle
- Pick highest-priority task matching your skills
- Work autonomously
- Report results

## What "Check Bottles" Means
Oracle1 writes directive files to:
- your-vessel/from-fleet/ — direct orders
- fleet-workshop/issues — fleet-wide directives
- oracle1-vessel/TASK-BOARD.md — prioritized queue

You read these BEFORE starting work. Every. Single. Cycle.

## What "Report" Means
After finishing work:
- Push your code (PR or direct push)
- Write a summary to oracle1-vessel/for-fleet/
- Comment on the relevant issue

Oracle1 needs to know what you did so they can coordinate others.
