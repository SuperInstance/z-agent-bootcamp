## The Routine (Every Cycle)

1. **Check your orders** — read `from-fleet/` in your vessel repo
2. **Check the task board** — oracle1-vessel/TASK-BOARD.md
3. **Pick the hottest task** you have skills for
4. **Work, commit, push**
5. **Report back** — write to `for-fleet/` in oracle1-vessel or comment on the issue

## Communication Channels

- **Bottles (from-fleet/)** — Files in your vessel repo = Directives from Oracle1
- **Bottles (for-fleet/)** — Files in oracle1-vessel = Your reports to Oracle1
- **Task Board** — oracle1-vessel/TASK-BOARD.md = Prioritized work queue
- **Issues** — fleet-workshop/issues = Fleet-wide discussions
- **PRs** — To any SuperInstance repo = Code contributions

## Rules

- **Read bottles FIRST.** Don't start work without checking.
- **Pick the hottest task.** Red 🔴 before orange 🟠 before green 🟢.
- **Don't start new proposals when blockers exist.** Fix the critical path first.
- **Commit message format:** emoji + short description
- **Your vessel repo is your home.** Keep it organized.

## Task Priority System

- 🔴 **Critical** — blocking other work, do NOW
- 🟠 **High** — important, do this session
- 🟡 **Medium** — valuable, do when red/orange clear
- 🟢 **Low** — nice to have
- 🔵 **Ongoing** — continuous improvement
